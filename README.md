# FAEAE UAV Path Planning Code Repository

This repository contains the MATLAB code used for the paper:

**Feasibility- and Adaptation-Enhanced Alpha Evolution for Constrained UAV Path Planning in Complex Low-Altitude Environments**

The code supports four groups of experiments:

1. CEC2017 benchmark tests
2. UAV parameter sensitivity analysis
3. Main UAV comparison experiments
4. UAV ablation experiments

> **Note**  
> The current paper version uses the `lite_v2` implementation of FAEAE.  
> In this README, **final paper version / no-fair-init version** means that the main comparison experiment does **not** impose the same reference-path initialization scheme on all compared algorithms.  
> This does **not** mean that the reference-path initialization module inside FAEAE is removed.

---

## 1. Basic Usage

Before running any experiment, open MATLAB in the repository root folder and run:

```matlab
clear; clc; close all;
addpath(genpath(pwd));
```

The repository is designed to run directly from the current folder.

Do **not** move files or reorganize folders unless all corresponding MATLAB paths are updated, because some scripts depend on the current file layout.

### 2.1 CEC2017 Benchmark Experiment

The CEC2017 experiment is used as a supplementary evaluation of general continuous optimization capability.

Run:

```matlab
clear; clc; close all;
addpath(genpath(pwd));
addpath(fullfile(pwd, 'cec2017_framework'));

cfg = getCEC2017Config('formal', 'high');
cfg.algorithms = {'AE', 'PSO', 'GWO', 'HHO', 'WOA', 'FAEAE'};

summary = run_cec2017_batch_lite(cfg);
```

Default configuration:

```text
Dimension  : 30
Functions  : F1 and F3--F30
Runs       : 30
Budget     : high, maxFEs = 300000
Algorithms : AE, PSO, GWO, HHO, WOA, FAEAE
```

Default result folder:

```text
results_cec2017_formal_high_YYYYMMDD_HHMMSS
```

Main output file:

```text
cec2017_batch_results.mat
```

The CEC summary is generated by:

```matlab
summary = summarize_cec2017_results(cfg.resultDir);
```

Useful table and figure generation scripts:

```matlab

cfg = getUAVComparisonConfig('formal');
```

## 6. Known Note

The script:

```text
run_faeae_lite_paper_pipeline.m
```

is a one-stop pipeline for CEC and UAV experiments. However, it calls:

```matlab
run_uav_comparison_lite_batch(uavCfg)
```

while the current final UAV main-comparison script is:

```matlab
run_uav_comparison_lite_v2_batch.m
```

Therefore, this pipeline may fail unless the function call is updated.

For safer reproduction, use the individual final scripts listed above.

## 7. Recommended Reproduction Order

For the final paper version, the recommended order is:

```matlab
% 1. CEC2017 benchmark
clear; clc; close all;
addpath(genpath(pwd));
addpath(fullfile(pwd, 'cec2017_framework'));
cfg = getCEC2017Config('formal', 'high');
summaryCEC = run_cec2017_batch_lite(cfg);

% 2. Parameter sensitivity
clear; clc; close all;
addpath(genpath(pwd));
summaryParam = run_uav_param_sensitivity_lite_v2();

% 3. Main UAV comparison
clear; clc; close all;
addpath(genpath(pwd));
cfgUAV = getUAVComparisonConfig('formal');
cfgUAV.useLiteFAEAE = true;
summaryUAV = run_uav_comparison_lite_v2_batch(cfgUAV);

% 4. UAV ablation
clear; clc; close all;
addpath(genpath(pwd));
summaryAblation = run_uav_ablation_lite_v2_batch();
```

After running the experiments, use the corresponding `make_*` or `export_*` scripts to generate paper tables and figures.

## 8. Notes for Future Maintenance

- The repository currently keeps both final and earlier versions of the code.
- The final paper version should be reproduced using the `lite_v2` scripts listed above.
- Result folders are kept mainly for record checking and figure/table reproduction.
- Avoid moving files unless the MATLAB path and function calls are updated accordingly.
- If a clean release is needed later, old scripts and result folders can be moved to an `archive` folder after confirming that all final scripts still run correctly.
